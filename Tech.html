Tech — 템플릿 상태 점검 가이드

Claude Code 멀티팀 협업 템플릿의 기술 상태를 확인하고, 표준 이탈 여부를 점검하는 기준 문서
PRD v1.3 PRD_dashboard v0.1 기준일 2026-07-07 브랜치 manager

1시스템 개요

이 템플릿은 폴더 = 권한, 문서 = 상태로 동작한다.

격리 + 팀 내 공유역할 세션은 공유 3폴더까지

기본은 시작 폴더 하위만. 역할 폴더의 .claude/settings.json이 00_Project·11_team_doc·90_result_output 접근을 허용. 타 역할·타 팀·루트는 금지

상속 INHERITANCE규칙은 내려오고, 데이터는 격리

CLAUDE.md가 상위→하위 자동 로드. 루트 「전 세션 공통 규칙」이 전 세션에 적용

문서가 곧 상태 DOCS AS STATE컨텍스트는 휘발된다

팀원은 handover_{역할}.md에 [NN_프로젝트명] 태그로 기록 → 팀 관리자가 process.md에 취합·반영 (전담)

root 명령 원칙 — *_root 명령은 전 팀 일괄 작업 후, 결과를 최종 관리자(루트) 문서 기준으로 취합·관리한다.

6불변식 — 깨지면 비표준 상태

1

팀 폴더명 ProjectTeam_{팀명} — 필수 4폴더 + CLAUDE.md 삭제·개명 금지

2

세션 폴더의 문서 요건 — 역할: CLAUDE.md+handover_{역할}.md+settings.json / 프로젝트 폴더엔 세션을 열지 않는다 (process.md 단독)

3

코드는 00_Project, process는 팀 관리자 — 코드 수정은 개발 역할·팀 관리자만, process.md 갱신은 팀 관리자 전담 (팀원은 [태그] 기록)

4

쓰기 주체 단일화 — 팀 대시보드=/report, 루트=/report_root, 90_result_output=패키지 역할. 손 편집 금지

5

명명 — 영문만(한글은 특이 경우·최단) · 날짜 YYYY-MM-DD · 결과물 YYYY-MM-DD_{팀명}_{제목}.md

6

분석 4역할은 분석 전용 — 파일 수정 금지. 별도 Builder 에이전트 없음 (팀 관리자 겸임)

7

양식 폴더는 팀이 아니다 — 보고·취합·대시보드 집계에서 제외

8

md가 진실의 원천 — HTML은 열람용 뷰. 반영할 내용은 handover/process/report에 쓴다
