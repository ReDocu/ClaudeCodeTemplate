# Claude Cockpit 사용 설명서 (초보자용)

여러 Claude Code 세션을 **한 화면에서 보고 조작**하는 로컬 대시보드입니다.
터미널마다 들어가서 확인할 필요 없이, 어떤 프로젝트의 어떤 세션이 **일하는 중인지 / 내 입력을 기다리는지**를 한눈에 파악하고, 필요할 때만 그 세션으로 점프합니다.

> **용어 두 개만 기억하세요**
> - **팀** = 프로젝트 하나 (wmux의 워크스페이스 하나)
> - **세션** = 팀 안에서 일하는 터미널/Claude 하나 (역할 이름을 붙여 구분: ops, frontend, qa …)

---

## 1. 준비물

| 항목 | 확인 방법 |
|------|-----------|
| Windows 11 | — |
| Node.js 20 이상 | 터미널에서 `node -v` |
| wmux (터미널 멀티플렉서) | 설치되어 있으면 됨 — 위치는 첫 실행 때 자동으로 찾음 |
| Claude Code | `claude` 명령이 동작하면 됨 |

## 2. 시작하기 — 클릭 한 번

**`ClaudeCockpit.exe`를 더블클릭**하세요. (실행이 막히면 `start.cmd`를 대신 실행)

자동으로 이 순서가 진행됩니다:

1. wmux가 꺼져 있으면 켭니다 (이미 켜져 있으면 그대로 사용)
2. 대시보드 서버를 띄웁니다 (검은 콘솔 창 = 서버. **닫아도 세션은 안 죽고**, 대시보드만 꺼집니다)
3. 등록된 팀들을 wmux에 맞춰 준비합니다
4. **기본 브라우저에 대시보드가 열립니다** (`http://127.0.0.1:7420/`)

- 몇 번을 다시 눌러도 안전합니다(이미 떠 있으면 재사용).
- 첫 실행 때 wmux 위치를 못 찾으면 콘솔에서 경로를 물어봅니다. 나중에 다시 설정하려면: `node teamctl/bin/teamctl.js boot --setup`
- 부팅하면서 **대시보드에 등록 안 된 옛 워크스페이스·세션을 정리**하고 싶으면: `node teamctl/bin/teamctl.js boot --clean` (더블클릭에도 적용하려면 `teamctl/workspace/config.json`에 `"cleanOnBoot": true` — exe 재빌드 불필요)

## 3. 화면 읽는 법

```
┌ 상단바 ─ ● live 배지 · usage 배지 · [⏻ 전체 핸드오버] [＋ 새 팀] ┐
├ 트리아지 존 ─ "지금 조치가 필요한 것"만 여기에 뜸                  │
│               (아무것도 없으면 "● 전부 정상" — 좋은 상태입니다)     │
├ 팀 카드들 ─ 팀 이름 · 폴더 경로 · git/포트 칩 · 세션 목록          │
│             각 세션 행을 클릭하면 오른쪽에 상세 드로어가 열림        │
└ 드로어 ─ 지금 하는 일 · 활동 피드 · 변경 파일 · 조작 버튼            ┘
```

### 세션 상태 기호 (색 + 모양 + 글자 3중 표기)

| 기호 | 뜻 | 내가 할 일 |
|------|-----|-----------|
| ⏸ 입력대기 | Claude가 **승인/입력을 기다림** | **가장 먼저 처리** — 트리아지 존에도 뜸 |
| ● 작업중 | Claude가 일하는 중 | 놔두면 됨 |
| ◆ 명령 대기 | Claude는 켜져 있고 한가함 | 시킬 일을 입력 |
| ❯ 터미널 | 아직 Claude를 안 켠 터미널 | 필요하면 ▶ Claude 실행 |
| ▲ 조치 필요 | 포트/서버가 죽어 있음 | 확인 필요 |

상단 **usage 배지**: `오늘 748k · 5h 631k ▏52M/186M · 남음 134M (72%)` 형식 — 오늘/최근 5시간 토큰 사용량과, 5시간 한도 대비 남은 양입니다. **주황(70%↑)/빨강(90%↑)** 이면 한도가 얼마 안 남은 것. (한도는 실측 추정치 — 공식 값 아님)

## 4. 기본 사용 흐름

### ① 팀 만들기
상단 **[＋ 새 팀]** → 팀 이름과 프로젝트 폴더 경로 입력 → 생성.
- wmux에 워크스페이스가 만들어지고, **배포·운영용 `ops` 세션이 자동으로 하나** 생깁니다(전용 폴더 `root/<팀>/ops/`에서 실행).
- 이미 그 프로젝트를 wmux에서 열어 둔 상태라면 새로 만들지 않고 **열려 있는 워크스페이스에 연결**됩니다.

### ② 세션 추가
팀 카드의 **[＋ 세션 스폰]** → 역할 이름 입력(예: frontend, qa) → 터미널 세션이 생깁니다.
- **같은 이름의 세션이 이미 열려 있으면 새로 만들지 않고 그 세션에 연결**됩니다("이미 열려있는 세션과 연결" 안내가 뜸).

### ③ Claude 시작
세션 행의 **[▶ Claude 실행]** 을 누르면 그 터미널에서 claude가 켜집니다.
- 이미 켜져 있으면 아무것도 다시 실행하지 않고 그대로 연결만 됩니다.
- 상태가 ❯ 터미널 → ◆ 명령 대기로 바뀌면 성공(실제 프로세스를 확인하므로 몇 초 걸릴 수 있음).

### ④ 관찰 — 세션에 들어가지 않고 판단
세션 행을 **클릭**하면 드로어가 열립니다:
- **지금** — 세션의 최근 발화(무슨 일을 하는 중인지)
- **활동 피드** — 최근 작업 내역
- **변경 파일** — git 기준 바뀐 파일과 +/− 줄 수
- **[📁 폴더]** — 작업 폴더를 탐색기로 열기 / 팀 카드의 git 칩(↗)은 원격 저장소 웹페이지 열기

### ⑤ 개입 — 필요할 때만 점프
드로어의 **[열기 ↗ (wmux 점프)]** 를 누르면 wmux의 해당 세션으로 이동합니다. 승인하거나 직접 명령을 친 뒤 대시보드로 돌아오면 됩니다.

### ⑥ 마무리
- **세션 하나 종료**: 드로어의 **[종료]** → 확인 창(미커밋 변경이 있으면 경고) → 종료 후 6초간 **[실행 취소]** 가능.
- **하루 마감**: 상단 **[⏻ 전체 핸드오버]** — 모든 Claude 세션이 각자 `handover.md`(인수인계 문서)를 갱신하고 `/exit`로 종료되어 터미널로 돌아갑니다. 진행 상황이 상단 배너에 표시됩니다. (승인 대기 중인 세션은 건너뜀)

## 5. 문제 해결 (FAQ)

**Q. 배지가 `○ demo`로 떠요 / 가짜 데이터가 보여요**
서버가 안 떠 있는 것입니다. `ClaudeCockpit.exe`를 다시 실행하세요. 화면의 **[▶ wmux 시작]** 버튼이 보이면 그걸 눌러도 됩니다.

**Q. 팀 카드에 세션이 하나도 없어요**
고장이 아니라 그 팀에 실행 중인 세션이 없는 것입니다. [＋ 세션 스폰]으로 추가하세요.

**Q. 드로어의 "지금/활동 피드"가 비어 있어요**
그 폴더에서 Claude를 아직 실행한 적이 없어서입니다. ▶ Claude 실행 후 첫 대화를 시작하면 채워집니다.

**Q. 상태가 바로 안 바뀌어요**
대시보드는 2.5초마다 갱신되고 일부 정보는 캐시를 씁니다. 몇 초 기다리면 반영됩니다.

**Q. wmux를 못 찾는다고 해요**
`node teamctl/bin/teamctl.js boot --setup` 을 실행해 wmux 설치 경로를 직접 지정하세요(한 번만 하면 저장됨).

**Q. 7420 포트가 이미 사용 중이래요**
`node teamctl/bin/teamctl.js serve --port 8080` 처럼 다른 포트로 띄울 수 있습니다.

**Q. 코드를 수정했는데 대시보드가 옛날처럼 동작해요**
서버가 수정 전 코드로 떠 있는 것입니다. 서버 콘솔 창을 닫고(또는 7420을 쓰는 node 프로세스 종료) exe를 다시 실행하세요.

**Q. 대시보드 창을 닫으면 세션이 죽나요?**
아니요. 대시보드/서버를 꺼도 wmux 세션들은 그대로 돕니다. 다시 exe를 누르면 이어서 관제됩니다.

**Q. 대시보드에 없는 옛 워크스페이스/세션이 wmux에 쌓여 있어요**
`node teamctl/bin/teamctl.js cleanup --dry` 로 뭐가 정리 대상인지 먼저 확인하고, 문제없으면 `--dry`를 빼고 실행하세요. 등록된 팀과 지금 명령을 실행 중인 창은 절대 건드리지 않습니다.

## 6. 안전 장치 (알아두면 안심)

- **로컬 전용** — 서버는 이 PC(127.0.0.1)에서만 접속되고 토큰 인증이 있습니다. 외부에서 못 들어옵니다.
- **파괴적 동작은 이중 안전** — 세션 종료는 확인 창 + 실행 취소, 미커밋 변경 시 경고.
- **자동 종료 없음** — 대시보드가 모르는(선언 안 된) 세션은 표시만 하고 절대 마음대로 끄지 않습니다. 정리는 사용자가 명시할 때만(`teamctl cleanup` / `boot --clean` / config `cleanOnBoot`).
- **중복 생성 없음** — 이미 열려 있는 것과 겹치는 팀/세션은 새로 만들지 않고 기존 것에 연결합니다.
- **시크릿 보호** — `.env` 값은 읽지도 표시하지도 않습니다(키 이름 개수만).
