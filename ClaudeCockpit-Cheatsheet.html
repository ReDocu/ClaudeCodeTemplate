ClaudeCockpit 치트시트

여러 Claude Code를 한 화면에서 관제 · 로컬 http://127.0.0.1:7420/

1시작

  • ClaudeCockpit.exe 더블클릭 (막히면 start.cmd) → 브라우저에 대시보드가 열림
  • 자동: wmux 켜기 → 서버 → 브라우저. 여러 번 눌러도 안전
  • 검은 콘솔 = 서버. 닫아도 세션은 안 죽음

2전체 흐름

＋ 새 프로젝트→ ▶ 활성화→ ＋ 세션 활성화→ ▶ Claude 실행
활성화는 방만 여는 것 — 세션(터미널)은 역할마다 따로 엽니다.

3세션 3단계 ★

○ 미연결세션 없음 → ＋ 세션 활성화 누름
? 확인 중연결 확인 중 (잠깐 대기)
❯ 준비됨↗ 세션 열기 ▶ Claude 실행
한 번에: 카드 아래 [＋ 모든 세션 활성화]. 이미 열린 건 재사용(중복 없음).

4활동 배지 = 지금 뭘 하나

⏳ 진행중작업 중 → 놔두기
⌛ 대기중지시 대기 → 일 시키기
⚠ 입력 대기승인 대기 → 먼저 처리
안 보이면: Claude 미실행 / 첫 프롬프트 전 / 훅 미설치.
훅 설치(1회): node cockpit/bin/activity-hook.mjs install

5프로젝트 상태 & 끄기

⏸ 대기중안 열림 → ▶ 활성화 / ▪ 아카이브
● 진행중열림 → 세션 활성화 · Claude 실행
▪ 종료됨보관 → 재개
끄기 두 가지(둘 다 확인 창): 세션 비활성화 하나 · 비활성화 전체. 되돌리기 없음.

6프로젝트 추가 3가지

＋ 새 프로젝트빈 폴더 새로 생성
기존 프로젝트 연동있는 폴더를 ops/로 이동 등록
＋ git 주소저장소를 ops/에 clone
git 저장소는 ops 폴더에만. 프로젝트 루트는 단순 폴더.

7문제 해결

▲ offline서버 죽음 → exe 재실행
세션 안 열림대개 wmux 꺼짐 → exe로 켜기
원인 보기node cockpit/bin/cockpit.js serve → 콘솔의 [wmux✗] 줄
포트 충돌serve --port 8080
옛 코드로 동작콘솔 닫고 exe 재실행 (디자인만 고쳤으면 새로고침)

8안전 장치

  • 로컬 전용 — 127.0.0.1 + 토큰. 외부 접속 불가
  • 자동 종료 없음 — 끄는 건 항상 내 버튼 + 확인
  • 중복 없음 — 열린 세션은 재사용
  • 시크릿 보호 — .env 값은 안 보여줌
