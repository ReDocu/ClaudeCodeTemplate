## 산정 기준

| 항목 | 기준 |
|---|---|
| **심각도** | 높음 3 / 중간 2 / 낮음 1 (인터뷰 기재값, 중복 통합 시 최고값) |
| **빈도** | 언급한 페르소나 수 (복수 언급 = 실사용에서 반복될 가능성 높음) |
| **해결 비용** | 낮음 = 문서 수정만 / 중간 = 명령·생성기 수정 / 높음 = 구조 변경·실측 필요 |
| **등급** | **P0** 즉시 (M6 시작 전) / **P1** M6 검증과 병행 / **P2** 다음 버전(v1.5) / **P3** 보류·모니터링 |

원칙: **조용한 데이터 유실 > 온보딩 이탈 > 운영 마찰 > 편의성** 순으로 가중. 심각도가 같으면 비용 낮은 것부터.
