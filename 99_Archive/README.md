## 이동 규칙 (모든 이동은 사용자 확인 후 수행)

| 대상 | 조건 | 이동 위치 |
|---|---|---|
| 종료된 프로젝트 | process 문서(`91_project_process/NN_xxx.md`) 제목에 `[종료]` 명시 | `teams/{팀명}/{프로젝트명}/` |
| 오래된 결과물 | `11_doc_result`에서 90일 경과 | `doc_result/{YYYY}/` |

- `/report_root` 실행 시 이동 후보를 대시보드의 「아카이브 후보」 섹션에 제안한다.
- 아카이브된 항목은 대시보드·INDEX 본문에서 제거한다. 템플릿 저장소가 추적하던 문서(process 문서·결과물 md)는 git 이력으로 추적 가능하다.

## 종료 프로젝트 이동 전 처리 (git 충돌 방지 — 필수)

프로젝트 작업물(`00_Project/NN_xxx/`)은 템플릿 저장소가 추적하지 않는다 (루트 `.gitignore` — 상태는 `91_project_process/`의 process 문서가 담당). 작업물 폴더를 **통째로** 여기로 옮기면 ignore 범위를 벗어나 **작업물 전체(중첩 `.git/` 포함)가 템플릿 저장소에 새로 추적된다** — 대용량 커밋·중첩 저장소 경고가 발생한다. 따라서:

1. **이동 대상은 process 문서(`91_project_process/NN_xxx.md`)와 최종 문서만** — 작업물(소스·에셋·빌드)은 프로젝트 자체 git 저장소에 남긴다. 남은 빈 작업 폴더(`00_Project/NN_xxx/`)는 확인 후 삭제한다.
2. 이동 전에 프로젝트 저장소의 **원격 URL·최종 커밋**을 process 문서 「진행 기록」에 기록한다.
3. 독립 저장소가 없는 소규모 프로젝트의 작업물 보존은 팀 `90_result_output/` 백업(패키지 역할)으로 대신한다.

**이동 후 분포**: 상태 문서(process 문서)는 여기 `teams/`, 최종 백업본은 팀 `90_result_output/`, 소스는 프로젝트 저장소. **팀 폴더 자체는 유지된다** (팀 단위 수명 관리는 F24 Lifecycle 팀 설계에서 다룰 예정 — `feature_changes.md` 참조).
