ClaudeCockpit Cheatsheet

Manage multiple Claude Code instances on one screen · Local http://127.0.0.1:7420/

1Getting started

  • Double-click ClaudeCockpit.exe (if blocked, use start.cmd) → the dashboard opens in your browser
  • Automatic: start wmux → server → browser. Safe to press multiple times
  • The black console = the server. Closing it does not kill sessions

2Overall flow

＋ New project→ ▶ Activate→ ＋ Activate session→ ▶ Run Claude
Activating only opens the room — sessions (terminals) are opened separately per role.

3Session in 3 steps ★

○ Not connectedNo session → press ＋ Activate session
? CheckingChecking connection (wait a moment)
❯ Ready↗ Open session ▶ Run Claude
All at once: [＋ Activate all sessions] below the card. Already-open ones are reused (no duplicates).

4Activity badge = what it's doing now

⏳ In progressWorking → leave it be
⌛ WaitingAwaiting instruction → give it work
⚠ Awaiting inputAwaiting approval → handle first
If not shown: Claude not running / before first prompt / hook not installed.
Install hook (once): node cockpit/bin/activity-hook.mjs install

5Project state & turning off

⏸ WaitingNot open → ▶ Activate / ▪ Archive
● In progressOpen → activate sessions · run Claude
▪ EndedArchived → resume
Two ways to turn off (both confirm): Deactivate session one · Deactivate all. No undo.

6Three ways to add a project

＋ New projectCreate a new empty folder
Link existing projectMove an existing folder into ops/ and register it
＋ git URLClone a repository into ops/
Git repositories go in the ops folder only. The project root is a plain folder.

7Troubleshooting

▲ offlineServer down → relaunch the exe
Session won't openUsually wmux is off → start it via the exe
See the causenode cockpit/bin/cockpit.js serve → the [wmux✗] line in the console
Port conflictserve --port 8080
Running old codeClose the console and relaunch the exe (if only the design changed, refresh)

8Safety features

  • Local only — 127.0.0.1 + token. No external access
  • No auto-shutdown — turning off is always your button + confirmation
  • No duplicates — open sessions are reused
  • Secret protection — .env values are not shown
