# 매뉴얼 — 팀별 관리자 (팀 관리자)

> 대상: 한 팀의 운영을 책임지는 사람. 역할 작업을 조율하고, 팀 보고·인수인계를 관리하며, **구현(Builder) 역할을 겸임**한다.

## 1. 당신은 누구인가

- **세션 위치**: `00_Team/ProjectTeam_{팀명}/` (팀 폴더)에서 `claude`를 실행한다.
- **권한**: 팀 폴더 안의 문서(팀 CLAUDE.md·handover.md·대시보드·팀 공용 문서)를 관리한다. 기획·패키지 등 **다른 역할의 작업 파일은 직접 수정하지 않는다** — 해당 역할 세션에 위임한다.
- **Builder 겸임**: 별도 구현 에이전트가 없다. 구현이 필요하면 팀 관리자가 `02_developer/`에서 수행한다.
- **격리**: 상위 폴더(루트·타 팀)에 접근하지 않는다. 전사 현황이 필요하면 사용자에게 요청한다.

## 2. 팀 폴더 구조 (필수 구성은 삭제·개명 금지)

```
ProjectTeam_{팀명}/
├── CLAUDE.md              팀 관리자 정의 (팀 목표는 생성 직후 작성!)   [필수]
├── handover.md            팀 인수인계 — /handover가 역할 handover 취합
├── 00_result_output/      팀 최종 결과물 (보고서·파이프라인 결과)      [필수]
├── 10_Dashboard/          팀 현황판 DASHBOARD.md (/report가 갱신)     [필수]
├── 11_team_doc/           팀 공용 문서 (회의록·결정 기록)              [필수]
├── 01_planner/            기획      ┐
├── 02_developer/          개발      ├ 역할 작업 공간 (표준 양식)
└── 03_package/            패키지    ┘
```

### 커스텀 역할이 필요할 때
표준(planner/developer/package)과 다른 구성이 필요하면 역할 폴더를 추가할 수 있다 (`NN_{역할}` 권장, 예: `04_designer`).
**직접 명령(세션)을 주는 폴더에는 반드시 두 파일을 만든다**:
1. `CLAUDE.md` — 역할 정의 (임무·권한·작업 규칙)
2. `handover_{역할}.md` — 역할 인수인계

## 3. 명령 레퍼런스

| 명령 | 하는 일 | 결과 |
|---|---|---|
| `/status` | 팀 대시보드 요약 조회 (갱신 안 함) | 출력만 |
| `/handover` | 역할 handover 취합 → 팀 인수인계 갱신 | `handover.md` |
| `/report` | 팀 정기 보고 + **팀 대시보드 갱신** | `00_result_output/report_{날짜}.md`, `10_Dashboard/DASHBOARD.md` |
| `/pipeline {주제}` | 4단계 분석 | `00_result_output/` |

## 4. 루틴

### 팀 생성 직후
```
1. CLAUDE.md의 「팀 목표」 작성
2. 작업 흐름 확인: 기획(01_planner) → 개발(02_developer) → 패키지(03_package)
3. 각 역할 담당자에게 역할 폴더에서 세션을 열도록 안내
```

### 조율 (수시)
```
1. 각 역할의 handover_{역할}.md 를 읽어 현황 파악
2. 역할 간 전달이 막힌 곳을 찾아 「다음 할 일」 조정
3. 구현이 필요하면 02_developer/ 에서 직접 수행 (Builder 겸임)
```

### 보고 (루트 /report_root 전, 권장 주 1회)
```
1. /report      → 보고서 + 팀 대시보드 갱신
2. /handover    → 팀 인수인계 최신화
3. 커밋: docs(report): {팀명} YYYY-MM-DD
```

## 5. 하면 안 되는 것

- 상위 폴더(루트·타 팀) 접근 — 격리 원칙. 필요한 정보는 사용자에게 요청.
- 필수 폴더(`00_result_output`/`10_Dashboard`/`11_team_doc`) 삭제·개명.
- 기획·패키지 작업 파일의 직접 수정 (구현 겸임은 `02_developer`에 한정).
- CLAUDE.md·handover 없는 폴더에서 세션을 열게 하는 것 — 커스텀 역할도 두 파일이 필수.
- 한글 폴더·파일명 (특이 경우에 한해 최대한 짧게).
