팀명: $ARGUMENTS
(비어 있으면 팀명을 물어본다.)

새 팀 작업 공간을 표준 구조로 생성한다.

## 1. 검증 (루트 전용)
- 현재 폴더에 `00_Team/`이 있는지 확인. 없으면 루트 전용임을 안내하고 중단.
- 팀명 검증: 영문/숫자/하이픈 권장. 한글·공백·특수문자가 포함되면 파일명·링크 호환성 리스크를 안내하고 **사용자 확인 후** 진행.
- 팀 폴더명은 `{메인폴더명}_{팀명}` (메인폴더명 = 루트 폴더 이름, `pwd`의 마지막 세그먼트).
- `00_Team/{팀폴더명}/`이 이미 존재하면 거부하고 중단.

## 2. 생성
1. `00_Team/{팀폴더명}/` 폴더 생성.
2. `90_Templates/CLAUDE.team.template.md`를 읽어 `{팀명}`·`{YYYY-MM-DD}`(오늘)를 치환 → `00_Team/{팀폴더명}/CLAUDE.md`.
3. `90_Templates/handover.template.md`를 읽어 대상명=`{팀명}`, 날짜=오늘로 치환하고, 현재 상태에 "팀 생성 직후 — 프로젝트 없음"을 기입 → `00_Team/{팀폴더명}/handover_{팀명}.md`.

## 3. 마무리
- 사용자에게 안내:
  - 팀 CLAUDE.md의 「팀 목표」를 작성할 것.
  - 팀 작업은 `00_Team/{팀폴더명}/`에서 새 세션을 열어 시작할 것 (격리 유지).
  - 첫 프로젝트는 팀 세션에서 `/new_project {프로젝트명}`으로 생성할 것.
- 커밋을 제안한다: `chore(team): {팀명} 생성`
