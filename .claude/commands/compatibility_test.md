프로젝트명: $ARGUMENTS
(비어 있으면 `00_Project/`의 폴더 목록을 보여주고 대상을 선택받는다.)

외부에서 가져왔거나 구버전 템플릿에서 옮겨온 프로젝트가 이 템플릿의 **데이터 계약**(명령·대시보드가 팀 데이터를 읽는 인터페이스)과 호환되는지 점검한다.
**점검은 읽기 전용** — 보정은 결과 보고 후 사용자가 승인한 항목만 수행한다.

## 1. 위치 판별 (팀 폴더 전용)
- `pwd`가 `.../00_Team/{팀폴더}`로 끝나고 `./00_Project/`가 있는지 확인. 아니면 팀 폴더에서 실행하라고 안내하고 중단.
- 대상 탐색: `00_Project/` 아래에서 {프로젝트명}과 일치하는 폴더 (`NN_` 접두가 아직 없어도 이름으로 찾는다).

## 2. 점검 항목 (순서대로 실행, 결과를 표로 수집)

| # | 항목 | 기준 (통과 조건) |
|---|---|---|
| 1 | 폴더명 | `NN_{프로젝트명}` 형식 — NN 2자리, 기존 번호와 충돌 없음, 이름은 영문/숫자/하이픈 |
| 2 | process 문서 존재 | `91_project_process/NN_{프로젝트명}.md`(프로젝트의 **유일한 상태 문서**)가 있는가 — 폴더명과 1:1 대응 |
| 3 | process 문서 구조 | 제목 `# Process — {이름} (갱신: YYYY-MM-DD)` + 필수 섹션: 목표/완료 조건 · 현재 단계(기획→개발→패키지, 진행 단계 굵게) · 상태 요약 표(상태등/진행률/담당/목표일) · 진행 기록(날짜 역순) · 다음 할 일 · 블로커 |
| 4 | 파싱 앵커 | 상태 요약 표에서 상태등·진행률을 기계적으로 추출 가능한가 (대시보드·/report가 읽는 값) · `[종료]`/`[보류]`는 제목 앞에만 표기 |
| 5 | 불변식 | 프로젝트 폴더 안에 `CLAUDE.md`·`handover*.md`·`process.md` **없음** — CLAUDE/handover는 폐지된 프로젝트 세션 모델의 잔재, 폴더 내 `process.md`는 구버전 계약의 잔재 (모두 `91_project_process/` process 문서로 통합·이전 대상) |
| 6 | git 경계 | 템플릿 저장소 기준 `git check-ignore` 표본 검사: `00_Project` 작업물은 전부 무시되고 **process 문서(`91_project_process/`)만 추적**되는가 · 자체 `.git/` 여부 확인 (독립 저장소면 정상 — 원격 URL을 process 문서에 기록했는지 확인) |
| 7 | 명명 위험 | 한글·공백이 든 파일/폴더명 표본 (경고만 — 특이 경우 외 영문 권장) |
| 8 | 태그 연동 (정보) | 역할 handover들에 `[NN_{프로젝트명}]` 태그 사용 여부 — 없으면 "아직 기록 없음"으로 표기 (실패 아님) |

## 3. 결과 보고
- 항목별 **✅통과 / ⚠경고 / ❌실패** 표 + 실패·경고마다 근거(파일 경로:줄)를 보여준다.
- 종합 판정: 전 항목 통과(정보 제외) → **"호환 — 바로 운영 가능"**. 실패 있음 → 아래 보정 계획을 제안.

## 4. 보정 (사용자 승인 항목만 수행)
- **process 문서 없음/구조 미달** → 표준 양식(`/new_project` 내장본과 동일)으로 `91_project_process/NN_{이름}.md` 생성·보완. 기존 자료(README·문서)에서 목표·현재 상태를 최대한 채우고, 진행 기록 첫 줄에 `{오늘}: 외부 프로젝트 반입 — /compatibility_test 보정`을 남긴다.
- **폴더명 형식 위반** → `NN_{이름}` 개명 제안 (NN = 기존 최대 번호 +1, process 문서 파일명도 함께).
- **CLAUDE.md·handover·폴더 내 process.md 잔존** → 내용을 `91_project_process/NN_{이름}.md`(현재 상태·다음 할 일)로 통합·이전한 뒤 삭제를 제안.
- **git 경계 위반** (작업물이 템플릿 저장소에 추적됨) → 원인을 보고하고 수동 조치를 안내한다 — **git 상태를 자동으로 바꾸지 않는다.**
- 보정한 항목은 재점검해 표를 갱신하고, 팀 관리자에게 process 문서 내용 확인을 안내한다.
- 커밋을 제안한다: `chore(project): {팀명}/{프로젝트명} 호환성 보정`
