프로젝트명: $ARGUMENTS
(비어 있으면 프로젝트명을 물어본다.)

팀 폴더 안에 새 프로젝트를 표준 구조로 생성한다.

## 1. 검증 (팀 폴더 전용)
- `pwd` 경로가 `.../00_Team/{팀폴더}`로 끝나는지 확인. 아니면 팀 폴더에서 실행하라고 안내하고 중단.
- 프로젝트명: 영문/숫자/하이픈 권장. 한글·공백 포함 시 리스크 안내 후 사용자 확인.
- `./{프로젝트명}/`이 이미 존재하면 거부하고 중단.

## 2. 생성 (양식은 여기 내장 — 하위 세션은 90_Templates 접근 불가)

`./{프로젝트명}/CLAUDE.md`:

```markdown
# CLAUDE.md — 프로젝트 관리자: {프로젝트명}

> 소속 팀: {팀명} · 생성일: {YYYY-MM-DD}
> 루트·팀 CLAUDE.md의 규칙이 이 폴더에도 적용된다.

## 프로젝트 목표
(이 프로젝트가 끝나면 얻는 것 — 생성 후 반드시 작성)

## 완료 조건 (Definition of Done)
- (무엇이 충족되면 "끝"인가 — 측정 가능하게)

## 현재 마일스톤
(지금 향하고 있는 지점 + 목표일)

## 작업 규칙
- 작업 파일은 모두 이 폴더 안에 둔다.
- 코드 수정은 이 프로젝트의 작업 세션이 수행한다 (역할 에이전트는 분석 전용).
- **세션을 마칠 때 `/handover`를 실행**해 `handover.md`를 갱신한다.
- 프로젝트가 종료되면 handover.md 상단에 `[종료]`를 명시한다 (아카이브 후보로 자동 인식됨).
```

`./{프로젝트명}/handover.md`: 표준 handover 양식(/handover 명령 참조)으로, 현재 상태에 "프로젝트 생성 직후 — 작업 전"을 기입해 생성.

## 3. 팀 문서 갱신
- 팀 `CLAUDE.md`의 「진행 중 프로젝트」 표에 새 프로젝트를 추가한다 (상태: ⚪ 대기, 시작일: 오늘).

## 4. 마무리
- 사용자에게 안내: 프로젝트 CLAUDE.md의 목표·완료 조건을 작성한 뒤 작업을 시작할 것.
- 커밋을 제안한다: `chore(project): {팀명}/{프로젝트명} 생성`
