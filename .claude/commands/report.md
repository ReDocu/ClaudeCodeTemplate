팀 정기 보고서를 생성한다.

## 1. 위치 확인 (팀 폴더 전용)
`pwd` 경로가 `.../00_Team/{팀폴더}`로 끝나는지 확인한다.
- 프로젝트 폴더라면: 팀 폴더에서 실행하라고 안내하고 중단.
- 루트라면: `/report_root`를 안내하고 중단.

## 2. 근거 수집
- 하위 각 프로젝트의 `handover.md`와 CLAUDE.md를 읽는다.
- `git log --oneline --since="14 days ago" -- .`으로 최근 활동을 확인한다.
- 팀 폴더의 직전 `report_*.md`(있다면)와 비교해 "지난 보고 이후" 변화를 파악한다.

## 3. 보고서 작성 → `./report_{YYYY-MM-DD}.md`

표준 양식 (원본: 90_Templates/report.template.md — 하위 세션 접근 불가로 여기 내장):

```markdown
# 정기 보고 — {팀명} ({YYYY-MM-DD})

## 상태 요약
| 항목 | 값 |
|---|---|
| 상태등 | 🟢 정상 / 🟡 지연·주의 / 🔴 블로킹 / ⚪ 대기·중단 (택1) |
| 전체 진행률 | {N}% |
| 다음 마일스톤 | {내용 + 목표일} |

## 3줄 요약
1. (지금 무엇을 하고 있는가)
2. (지난 보고 이후 무엇이 됐는가)
3. (무엇이 문제인가 / 없으면 "블로커 없음")

## 최근 성과
- (완료 항목 + 결과물 링크)

## 블로커 / 크로스팀 이슈
- (막힌 것 — 누가/무엇이 필요한지, 언제부터. 없으면 "없음")

## 프로젝트별 상세
| 프로젝트 | 상태 | 진행률 | 비고 |
|---|---|---|---|
```

- 상태등 판정 기준: 블로커 존재 → 🔴, 마일스톤 지연 또는 7일 이상 활동 없음 → 🟡, 그 외 진행 중 → 🟢, 중단·대기 → ⚪.
- 진행률은 프로젝트 handover의 완료/잔여 항목에서 추정하고, 추정임을 비고에 남긴다.

## 4. 마무리
- 이 보고서는 루트의 `/report_root`가 수거해 대시보드에 반영함을 안내한다.
- 커밋을 제안한다: `docs(report): {팀명} {YYYY-MM-DD}`
