팀 정기 보고서를 생성하고 팀 대시보드를 갱신한다.

## 1. 위치 확인 (팀 폴더 전용)
`pwd` 경로가 `.../00_Team/{팀폴더}`로 끝나는지 확인한다.
- 역할 폴더(`01_planner` 등)나 프로젝트 폴더라면: 팀 폴더에서 실행하라고 안내하고 중단.
- 루트라면: `/report_root`를 안내하고 중단.

## 2. 근거 수집
- 각 역할 폴더(필수 폴더 `00_result_output`/`10_Dashboard`/`11_team_doc` 제외 — 표준: `01_planner`/`02_developer`/`03_package`, 커스텀 포함)의 `handover_{역할}.md`와 CLAUDE.md, 역할 하위 프로젝트의 `handover.md`를 읽는다.
- `git log --oneline --since="14 days ago" -- .`으로 최근 활동을 확인한다.
- `./00_result_output/`의 직전 `report_*.md`(있다면)와 비교해 "지난 보고 이후" 변화를 파악한다.

## 3. 보고서 작성 → `./00_result_output/report_{YYYY-MM-DD}.md`

표준 양식 (원본: 90_Templates/report.template.md — 하위 세션 접근 불가로 여기 내장):

```markdown
# 정기 보고 — {팀명} ({YYYY-MM-DD})

## 상태 요약
| 항목 | 값 |
|---|---|
| 상태등 | 🟢 정상 / 🟡 지연·주의 / 🔴 블로킹 / ⚪ 대기·중단 (택1) |
| 전체 진행률 | {N}% |
| 다음 마일스톤 | {내용 + 목표일} |

## 3줄 요약
1. (지금 무엇을 하고 있는가)
2. (지난 보고 이후 무엇이 됐는가)
3. (무엇이 문제인가 / 없으면 "블로커 없음")

## 최근 성과
- (완료 항목 + 결과물 링크)

## 블로커 / 크로스팀 이슈
- (막힌 것 — 누가/무엇이 필요한지, 언제부터. 없으면 "없음")

## 역할별 상세
| 역할 | 상태 | 진행률 | 비고 |
|---|---|---|---|
| 기획 (01_planner) | 🟢 | {N}% | |
| 개발 (02_developer) | 🟢 | {N}% | |
| 패키지 (03_package) | ⚪ | {N}% | |
```

- 상태등 판정 기준: 블로커 존재 → 🔴, 마일스톤 지연 또는 7일 이상 활동 없음 → 🟡, 그 외 진행 중 → 🟢, 중단·대기 → ⚪.
- 진행률은 역할 handover의 완료/잔여 항목에서 추정하고, 추정임을 비고에 남긴다.
- 역할 하위 프로젝트가 있으면 해당 역할 행 아래에 `└ {프로젝트명}` 행으로 추가한다.

## 4. 팀 대시보드 갱신 → `./10_Dashboard/DASHBOARD.md` (덮어쓰기)
- 「상태 요약」: 보고서의 상태 요약 표를 옮긴다.
- 「역할별 현황」: 역할별 상세 표 + 각 `handover_{역할}.md`의 최근 갱신일.
- 「블로커 / 크로스팀 이슈」: 보고서의 블로커 섹션.
- 「최근 결과물」: `./00_result_output/` 최신 5건.
- 상단 갱신일을 오늘로.

## 5. 마무리
- 이 보고서·대시보드는 루트의 `/report_root`가 수거해 루트 대시보드에 반영함을 안내한다.
- 커밋을 제안한다: `docs(report): {팀명} {YYYY-MM-DD}`
