description: 현재 폴더 기준 인수인계 문서 생성/갱신 (프로젝트/팀 폴더에서 실행)
---

현재 폴더의 인수인계 문서를 생성·갱신한다. 아래 절차를 따르라.

## 1. 위치 판별
`pwd`로 현재 경로를 확인한다.
- 경로가 `.../00_Team/{팀폴더}/{프로젝트명}` (00_Team 아래 두 단계) → **프로젝트 모드**
- 경로가 `.../00_Team/{팀폴더}`로 끝남 → **팀 모드**
- 현재 폴더에 `00_Team/`과 `10_Dashboard/`가 존재 → 루트다. `/handover_root`를 사용하라고 안내하고 **중단**.
- 어느 것도 아니면 사용자에게 위치를 확인하고 중단.

## 2. 근거 수집
- 기존 인수인계 문서(있다면)와 현재 폴더의 CLAUDE.md를 읽는다.
- `git log --oneline -20 -- .`으로 최근 변경을 확인한다.
- 현재 폴더의 작업 파일 목록과, 이번 세션에서 수행한 작업 내용을 근거로 삼는다.
- **추측으로 채우지 않는다.** 확인 안 되는 항목은 "(미확인)"으로 남긴다.

## 3. 문서 작성 (아래 표준 양식, 덮어쓰기)
- **프로젝트 모드** → `./handover.md`
- **팀 모드** → 하위 각 프로젝트의 `handover.md`를 모두 읽고, **요약 + 상대경로 링크**로 취합해 `./handover_{팀명}.md` 작성 (원문 통째 복사 금지). 프로젝트에 handover가 없으면 "미작성"으로 표기.

표준 양식 (원본: 90_Templates/handover.template.md — 하위 세션 접근 불가로 여기 내장):

```markdown
# Handover — {대상명} (갱신: {오늘 날짜 YYYY-MM-DD})

## 목표
(이 프로젝트/팀이 끝나면 얻는 것 — 1~2문장)

## 현재 상태
(한 문단 요약 — 새로 온 사람이 이것만 읽어도 감이 잡히게)

## 완료된 것
- (완료 항목 — 결과물·파일 경로 포함)

## 진행 중인 것
- (항목 — 마지막 작업 시점, 어디까지 됐는지)

## 다음 할 일 (우선순위 순)
1. (가장 급한 것)

## 주의사항 / 함정
- (다음 사람이 반드시 알아야 할 것 — 이유 포함)

## 핵심 파일 경로
- `경로` — 역할 한 줄
```

팀 모드는 위 양식에 `## 프로젝트별 요약` 섹션을 추가하고, 프로젝트마다 3줄 요약 + `[handover](./{프로젝트}/handover.md)` 링크를 넣는다.

## 4. 마무리
- 갱신된 문서의 3줄 요약을 사용자에게 보여준다.
- 커밋을 제안한다: `docs(handover): {대상명}`
