주제: $ARGUMENTS
(주제가 비어 있으면 무엇을 분석할지 사용자에게 묻고 시작한다.)

Explorer → Educator → Critic → Advisor 4단계 분석을 수행한다.

## 0. 위치 판별
`pwd`로 현재 위치를 확인한다.
- **루트**(현재 폴더에 `00_Team/`·`10_Dashboard/` 존재): 중간 산출물 → `0X_{역할}/output/{YYYY-MM-DD}_{주제}_{역할}.md`, 최종 결과물 → `11_doc_result/{YYYY-MM-DD}_{팀명 또는 root}_{주제}.md` + `INDEX.md` 갱신.
- **팀 폴더**(경로가 `.../00_Team/{팀폴더}`로 끝남): 중간 산출물 → `./_pipeline/{YYYY-MM-DD}_{주제}_{역할}.md`, 최종 결과물 → `./11_team_doc/{YYYY-MM-DD}_{팀명}_{주제}.md` (문서 취합 폴더).
- **그 외**(역할 폴더): 중간 산출물 → `./_pipeline/{YYYY-MM-DD}_{주제}_{역할}.md`, 최종 결과물 → `./{YYYY-MM-DD}_{팀명}_{주제}.md` 또는 공유 `../11_team_doc/`. 타 역할 폴더에는 접근하지 않는다.

## 1~4단계 (순차 실행 — 각 단계는 이전 단계 출력을 입력으로 받는다)

1. **explorer** 서브에이전트 실행. 입력: 주제 + 분석 대상 경로. 출력: 탐색 보고.
2. **educator** 서브에이전트 실행. 입력: 주제 + 탐색 보고 전문. 출력: 설명 문서.
3. **critic** 서브에이전트 실행. 입력: 주제 + 탐색 보고 + 설명 문서. 출력: 검증 보고(판정 포함).
4. **판정 처리**:
   - **승인 / 조건부 승인** → 5단계로.
   - **반려** → Critic이 지목한 단계(탐색 또는 설명)를 지적 사항 전문과 함께 **1회만** 재실행하고, critic으로 재검증한다. **재순환은 최대 1회.** 재반려되면 중단하지 말고 반려 사유를 최종 결과물에 그대로 기록한다 (실패도 결과다).
5. **advisor** 서브에이전트 실행. 입력: 주제 + 설명 문서 + 검증 보고(조건 포함). 출력: 권고 문서.

각 단계가 끝날 때마다 중간 산출물을 0단계에서 정한 위치에 저장하고, 사용자에게 한 줄 진행 상황을 알린다.

## 5. 최종 결과물 작성

표준 양식 (원본: 90_Templates/result.template.md — 하위 세션 접근 불가로 여기 내장):

```markdown
# {주제}

| 항목 | 내용 |
|---|---|
| 날짜 | {YYYY-MM-DD} |
| 팀 | {팀명 또는 root} |
| 유형 | 분석(파이프라인) |
| 생성 경로 | /pipeline |

## 요약 (3줄)

## 본문
(Educator 설명 문서의 핵심)

## 검증 결과 (Critic)
- 판정: {승인/조건부 승인/반려}
- 주요 지적과 반영 내역: (재순환이 있었다면 무엇이 고쳐졌는지)

## 권고 (Advisor)

## 근거 / 참고 경로

## 부록
- 중간 산출물: (각 단계 파일 링크)
```

## 6. 마무리
- 최종 결과물의 「요약 3줄 + 판정 + 권고 한 줄」을 사용자에게 보여준다.
- 커밋을 제안한다: `docs(result): {주제}`
