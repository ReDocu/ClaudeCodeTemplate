2Three Steps to Open a Session — most important

Activating (▶ Activate) just opens the room; it does not create a session (terminal). You open sessions one at a time from each role row.

Session rows inside an in-progress card — the buttons change with the state
○Not connected ops ops pinnedNo session — activation needed
＋ Activate session
❯Terminal devChecking connection…
? Checking connection
❯Terminal artroot/MyApp/art/↗
↗ Open session▶ Run Claude
  1. ① Not connected — there is no terminal yet. Pressing ＋ Activate session creates that role's terminal in wmux.
  2. ② Checking connection — wait a moment. During this time the Run Claude button does not appear yet (to prevent misfires).
  3. ③ Ready — use ↗ Open session to jump to that terminal, and ▶ Run Claude to turn Claude on.
To open everything at once, use [＋ Activate all sessions] below the card. Sessions that are already open are reused rather than recreated (no duplicates).

4Session Drawer — Click for Details

Click a session row to open a detail drawer on the right.

Session detail drawer
MyApp / ops✕
✦claude running⌛awaiting response 📌 ops pinned
Open ↗ (wmux jump)📁 Folder ▶ Run ClaudeDeactivate session
  1. Status · activity badges — check claude on/off, in progress/waiting, whether ops is pinned, and the working folder (cwd).
  2. Open ↗ (wmux jump) — move to that terminal to approve or run commands directly. 📁 Folder — open the folder in the file explorer.
  3. Deactivate session — end just this one session (red). If Claude is working, a confirmation dialog appears.

⚠️ There is no undo. That's why every turn-off button goes through a confirmation dialog. The only ways to turn things off are [Deactivate session] (one) and [Deactivate] (all) — cockpit never turns anything off on its own.
