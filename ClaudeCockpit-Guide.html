2세션을 여는 3단계 — 가장 중요

활성화(▶ 활성화)는 방을 여는 것일 뿐, 세션(터미널)은 만들지 않습니다. 세션은 각 역할 행에서 하나씩 엽니다.

진행중 카드 안의 세션 행 — 상태에 따라 버튼이 달라집니다
○미연결 ops ops 고정세션 없음 — 활성화 필요
＋ 세션 활성화
❯터미널 dev연결 확인 중…
? 연결 확인 중
❯터미널 artroot/MyApp/art/↗
↗ 세션 열기▶ Claude 실행
  1. ① 미연결 — 아직 터미널이 없습니다. ＋ 세션 활성화 를 누르면 그 역할의 터미널이 wmux에 생깁니다.
  2. ② 연결 확인 중 — 잠깐 기다립니다. 이때는 아직 Claude 실행 버튼이 안 나옵니다(오발동 방지).
  3. ③ 준비됨 — ↗ 세션 열기 로 그 터미널로 점프하고, ▶ Claude 실행 으로 Claude를 켭니다.
한 번에 다 열려면 카드 아래 [＋ 모든 세션 활성화]. 이미 열린 세션은 새로 만들지 않고 재사용합니다(중복 없음).

3Claude가 지금 뭘 하는지 — 활동 배지

Claude가 켜진 세션에 붙는 배지로, 들어가지 않고도 무엇을 할지 압니다.

세션 행의 활동 배지
✦claude⏳진행중 devroot/MyApp/dev/
↗ 세션 열기
✦claude⌛대기중 ops ops 고정root/MyApp/ops/
↗ 세션 열기
✦claude⚠입력 대기 artroot/MyApp/art/
↗ 세션 열기
배지뜻내가 할 일
⏳진행중Claude가 작업하는 중놔두면 됩니다
⌛대기중응답을 마치고 다음 지시 대기시킬 일을 입력
⚠입력 대기승인·권한 입력을 기다림먼저 처리
배지가 안 보이면: Claude를 아직 안 켰거나(❯터미널), 첫 프롬프트 전이거나, 활동 훅 미설치입니다. 훅 설치(최초 1회): node cockpit/bin/activity-hook.mjs install

4세션 서랍 — 클릭해서 자세히 보기

세션 행을 클릭하면 오른쪽에 상세 서랍이 열립니다.

세션 상세 드로어
MyApp / ops✕
✦claude 실행중⌛응답 대기중 📌 ops 고정
열기 ↗ (wmux 점프)📁 폴더 ▶ Claude 실행세션 비활성화
  1. 상태·활동 배지 — claude on/off, 진행중/대기중, ops 고정 여부, 작업 폴더(cwd)를 봅니다.
  2. 열기 ↗ (wmux 점프) — 그 터미널로 이동해 직접 승인·명령. 📁 폴더 — 탐색기로 폴더 열기.
  3. 세션 비활성화 — 이 세션 하나만 종료(빨강). Claude가 일하는 중이면 확인 창이 뜹니다.
