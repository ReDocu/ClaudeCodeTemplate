# 인사이트 & 핵심 전략 (가설) — Claude Cockpit

**작성일**: 2026-07-10
**입력 자료**: `01-interview.csv` (참가자 8명 · 응답 47건), `02-interview-summary.md`
**성격**: 정성 리서치(n=8) 기반 **전략 가설**. 정량 검증 전이므로 각 항목에 확신도(높음/중간/낮음)를 표기함.
**한 줄 명제**: *이름은 "조종석(Cockpit)"인데, 제품은 아직 "계기판(Dashboard)"이다.* 전략의 본질은 **관제탑에서 조종석으로** 넘어가는 것.

## 0. 전략 명제 (North Star)

> **"정직하게 살아있는 화면 위에서, 이탈 없이 판단하고 행동하는 단일 통제 표면이 된다."**

세 개의 기둥으로 지탱한다.

| 기둥 | 정의 | 왜 지금 | 뿌리 인사이트 |
|------|------|---------|----------------|
| **① 정직한 라이브니스** | 화면이 살아있는지/죽었는지를 절대 속이지 않는다 | 신뢰가 없으면 아무도 이 화면으로 세션을 죽이지 않는다 | 인사이트 2 |
| **② 이탈 없는 통제** | 모든 신호가 그 자리에서 행동으로 이어진다 (관찰→행동 루프 닫기) | "보이는데 못 누른다"가 전 페르소나 공통 벽 | 인사이트 1·5 |
| **③ 모두를 위한 명료함** | 색·포커스·피드백을 이중/삼중으로 부호화한다 | 접근성 결함 = 일반 사용자도 겪는 UX 부채 | 인사이트 3 |

수익화 쐐기(wedge)는 별도 축: **프리랜서 청구 롤업**(인사이트 4).

## 1. 심층 인사이트 5 (발견 → 패턴 → 전략적 함의)

### 인사이트 1 — 제품이 "보여주기"에서 멈춘다: 관찰→행동 루프가 끊겼다 · 확신도 **높음**
- **패턴**: 서로 다른 5명이 같은 벽에 부딪힘. 입력대기 카운트를 눌러도 무반응(F2·P01), 대기 승인 버튼 없음(F8·P04), 인수인계는 읽기 전용(F6·P05), 오류에 복구 액션 없음(F21·P04), 대기 해결 위치 불명(F8/F12·P03).
- **역설**: 유일하게 "한 방에 되는" 액션이 **가장 위험한 종료(F5)** 다 — 확인도 undo도 없이. 즉 **안전한 행동은 없고, 위험한 행동만 원클릭**이라는 뒤집힌 액션 모델.
- **전략적 함의**: 신규 기능을 더 얹는 게 아니라 **이미 보여주는 모든 신호를 "문(door)"으로 바꾸는 것**이 방향. 카운트→내비게이션, 대기→인라인 승인, 인수인계→수락, 오류→인라인 복구. 이것이 "조종석" 정체성의 핵심.

### 인사이트 2 — 신뢰가 세 겹으로 깨져 있고, 신뢰 없이는 조종석이 성립하지 않는다 · 확신도 **높음**
- **패턴(3중 균열)**: ① 죽은 대시보드를 살아있다고 믿음(F4, 심각도 5 — 전체 최악), ② "라이브"라 써놓고 실제론 트랜스크립트(F19), ③ 보안 태세(로컬·잠김)가 화면에 안 보임(F20).
- **가장 무서운 문장**(P01): *"서버가 죽어도 저 숫자는 계속 오른다면 멈춘 화면 보고 판단하는 거잖아요."* — **죽은 계기판이 살아있는 인프라에 대한 오판을 유발**한다.
- **전략적 함의**: 이 제품은 DB·마이그레이션 세션을 죽이고 살리는 통제 화면(P04·P05). 신뢰는 **폴리시가 아니라 운영 면허(license to operate)** 다. 신뢰 적자 → 사용자가 행동을 안 함 → 관찰→행동 루프를 아무리 만들어도 안 쓰임. **정직한 라이브니스는 확장보다 먼저다.**

### 인사이트 3 — 접근성은 규정준수가 아니라 "전체 UX의 카나리아"이자 시장이다 · 확신도 **중간**
- **패턴**: 접근성 의존 사용자 3명(P06·P07·P08 = 표본의 37.5%)이 심각도 5 블로커를 사실상 독식(포커스 가로챔 F1, 파괴적 무음 F11/F5, 대기 침묵 F12).
- **결정적 통찰**: **같은 결함이 일반 사용자도 무너뜨린다.** 정상 시력 P01이 색상 단독 신호를 놓쳤고(빨간 점이 노란 테두리에 묻힘 F3), 정상 시력 P03이 야외에서 저대비 텍스트를 못 읽음(F13). 색 단독·포커스 방치·무음 피드백은 **엣지 조건의 일반 사용자에게도 실패**한다.
- **전략적 함의**: 접근성을 비용에서 **레버리지**로 재정의. 이중 부호화(색+형태+텍스트)·포커스 규율·라이브 리전은 중앙값 사용자의 품질까지 끌어올린다. **"접근성 우선 = 품질 우선."** 동시에 이 표본 기준 잠재시장의 37.5%.

### 인사이트 4 — 유일하게 명확한 지불 의사는 "프리랜서 청구"다 → 이것이 쐐기다 · 확신도 **중간**
- **패턴**: P01과 P05가 **독립적으로** 같은 말 — *"클라이언트별 청구 롤업 되면 유료라도 쓴다"*(M4). 전체 데이터에서 유일하게 명시된 지불 의사.
- **거의 다 왔다**: 세션별 비용은 이미 나오고(P01), 세션별 diff는 이미 **청구 근거**로 인정받음(P05 강점 — *"클라이언트한테 근거로 쓰기 좋아요"*). 딱 마지막 한 뼘 — **클라이언트 단위 묶기** — 이 없다.
- **JTBD**: *"고객에게 근거 기반으로 시간을 청구한다."* diff 가시성이 이미 여기서 해자(moat) 역할.
- **전략적 함의**: **프리랜서-다클라이언트**가 교두보 세그먼트, **청구 롤업**이 수익화 훅. 번들: 정직한 근거(diff+비용) → 클라이언트 롤업 → 인보이스/리포트.

### 인사이트 5 — 강점은 "드로어" 하나에 응축돼 있다 → 전략은 여기서 확장한다 · 확신도 **높음**
- **패턴**: 드로어를 P01·P03·P05가 각각 칭찬 — *"안 들어가도 판단이 된다."* 이것이 차별화된 실제 가치 = **이탈 없는 상황 인식(situational awareness without context-switch).**
- **기반도 이미 있다**: 키보드 토대 존재(P06 강점), 올바른 라벨 패턴 일부 존재(P07 강점), 텍스트 병기 패턴 일부 존재(P08 강점). 뼈대는 좋다.
- **전략적 함의**: **드로어를 "행동 가능한 주 표면(control panel)"으로 승격.** 승인/안전한 종료/복구/수락 버튼을 드로어 안에 넣으면 → ⓐ 관찰→행동 루프를 닫고(인사이트 1), ⓑ 검증된 강점 위에 쌓고, ⓒ 위험한 점프를 회피한다. **드로어 = 조종석의 조종반.**

## 2. 전략적 베팅 (Strategic Bets)

> 각 베팅 = 가설 + 근거 + 성공의 모습 + 최대 리스크.

### 베팅 A — "정직한 라이브니스"를 먼저 배송한다 (기둥 ①) · 우선순위 **최상**
- **가설**: 화면이 죽었을 때 죽었다고 말하면(스탬프·stale 배지·실제 소스 라벨) 사용자는 비로소 화면으로 행동하기 시작한다.
- **근거**: F4(심각도 5, 최악 시나리오)·F19·F20. 신뢰 적자가 행동을 막는 상류 원인.
- **성공의 모습**: 서버 단절 후 N초 내 stale 전환, "라이브/트랜스크립트/n초 전(정지)" 정직 라벨, 보안 태세 배지 상시 노출.
- **최대 리스크**: stale 임계값을 너무 민감하게 잡으면 "늑대 소년" 피로. → 계층별(서버/세션/MCP) 임계값 실험 필요.

### 베팅 B — 드로어를 조종반으로 만든다 = 관찰→행동 루프 닫기 (기둥 ②) · 우선순위 **최상**
- **가설**: 신호를 문으로 바꾸면(카운트→점프, 대기→인라인 승인, 인수인계→수락, 오류→복구) 세션 진입 횟수와 트리아지 시간이 급감한다.
- **근거**: F2/F30·F8·F6·F21·F5. 전 페르소나 공통 벽 + 이미 칭찬받는 드로어 위에 얹음.
- **성공의 모습**: 승인/거절·안전한 종료(확인+undo)·인수인계 accept·인라인 로그인이 **드로어 안에서** 완결. 세션 직접 진입은 예외가 됨.
- **최대 리스크**: 파괴적 액션의 오클릭(F5). → 확인 + undo + 위험 액션 시각 분리를 동반 배송.

### 베팅 C — 접근성 우선을 품질 전략으로 재프레이밍 (기둥 ③) · 우선순위 **높음**
- **가설**: 이중 부호화 + 포커스 규율 + 라이브 리전을 깔면 접근성 사용자뿐 아니라 일반 사용자 트리아지 정확도도 오른다.
- **근거**: 심각도 5의 3/4가 a11y(F1·F11·F12) + 같은 결함이 정상 시력도 실패시킴(F3·F13).
- **성공의 모습**: 색+형태+텍스트 삼중 상태, 열기 시 드로어로 포커스 이동·닫기 시 복귀, 파괴적/대기 전이를 라이브 리전으로 호명.
- **최대 리스크**: "규정준수 체크박스"로 축소 해석되어 우선순위가 밀리는 것. → 일반 사용자 지표(놓친 오류율)로 정당화.

### 베팅 D — 프리랜서 청구를 수익화 쐐기로 (별도 축) · 우선순위 **다음 분기**
- **가설**: 클라이언트별 청구 롤업이 무료→유료 전환의 첫 훅이 된다.
- **근거**: M4 — 유일한 명시적 지불 의사(P01·P05). diff = 이미 인정된 청구 근거.
- **성공의 모습**: 세션 비용 → 클라이언트 롤업 → 내보내기(인보이스/리포트) 흐름 완결. 유료 전환 의향 정량 확인.
- **최대 리스크**: n=2의 지불 의사를 과대 해석. → 가격 민감도·롤업 목업 반응을 먼저 검증(아래 실험 E3).

## 4. 우선순위 로드맵 (전략 기둥 정렬)

> 요약본의 P0~로드맵을 **전략 서사**로 재배열. Now = 운영 면허 확보, Next = 조종석 완성, Later = 수익화·확장.

### NOW — 신뢰와 안전 (출시 차단 급)
- **정직한 라이브니스**: stale 감지·정지 스탬프·실제 소스 라벨 (F4·F19)
- **접근성 치명 3종**: 포커스 가로챔 제거(F1)·파괴적 무음 해소(F11/F5)·대기 호명(F12)
- **파괴적 액션 안전화**: 확인 + undo + 위험 액션 시각 분리 (F5)

### NEXT — 조종석 완성 (관찰→행동 루프)
- 카운트→세션 내비게이션 + 신호등 최악-자식 롤업 (F2/F30·F3)
- 드로어 인라인 액션: 승인/거절·인수인계 accept·오류 복구 (F8·F6·F21)
- 인박스 버튼 시맨틱 + 포커스 이동/복귀 (F10·F9)
- 상태 이중 부호화 + 범례 형태 키 (F14/F15) · 저대비 상향(F13)

### LATER — 활성화 & 수익화
- 온보딩: 제로-팀 빈 상태 + 복사 가능 명령, CLI/대시보드 분리 교육 (F23·F24)
- 보안 태세 배지, MCP 헬스 노출 (F20·F25/D15)
- **프리랜서 청구 롤업 → 내보내기** (M4) ← 수익화 훅

## 6. 핵심 가정 & 저비용 검증 실험

> 이 전략은 n=8 정성에 기댄 **가설**이다. 가장 값비싼 가정부터 싸게 깨본다.

- **E1 (신뢰가 상류 원인인가)**: stale 라벨 유무 A/B — "화면으로 종료를 실행하는 비율"이 오르나? *가정: 신뢰→행동.*
- **E2 (드로어 액션이 진입을 대체하나)**: 드로어에 승인 버튼 프로토타입 → 세션 직접 진입 감소 측정. *가정: 관찰→행동 루프.*
- **E3 (청구 지불 의사가 n=2 이상인가)**: 롤업 목업 + 가격 제시로 프리랜서 세그먼트 전환 의향 정량화. *가정: M4 = 수익화 훅.*
- **E4 (a11y 수정이 일반 사용자도 돕나)**: 이중 부호화 전/후 정상 시력 사용자의 "놓친 down 포트" 비율. *가정: 접근성 = 품질 레버리지.*
- **E5 (온보딩 절벽이 활성화를 막나)**: 제로-팀 빈 상태 + 복사 명령 유무로 TTFV 비교. *가정: 첫인상이 채택 관문.*

## 7. 한 문단 요약 (경영진용)

Claude Cockpit은 **"보여주기"는 이미 잘하지만 "행동하기"에서 멈춘 계기판**이다. 8명 인터뷰의 일관된 서사는 하나 — *보이는데 그 자리에서 누를 수가 없다.* 이를 풀 열쇠는 이미 칭찬받는 **드로어**를 조종반으로 승격해 관찰→행동 루프를 닫는 것이며, 그 전제는 **화면이 죽었을 때 죽었다고 말하는 정직한 라이브니스**(운영 면허)다. 접근성 결함은 규정 문제가 아니라 일반 사용자도 겪는 품질 부채의 카나리아이므로 **품질 우선 전략**으로 병행한다. 수익화는 유일하게 명확한 지불 의사인 **프리랜서 청구 롤업**을 쐐기로 삼는다. 순서는 **신뢰 → 행동 → 활성화 → 과금**. 모든 항목은 위 저비용 실험(E1~E5)으로 검증 후 확대한다.
