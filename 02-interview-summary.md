## 주제별 핵심 요약

### 1. 트리아지 — "지금 개입할 곳"을 못 찾는다
- **입력대기 카운트가 죽은 버튼** (P01, F2/F30): 상단 "입력대기 1"을 눌러도 반응이 없어 카드를 눈으로 훑게 됨. 팀이 커지면 못 버팀.
- **신호등이 최악 자식을 안 알림** (P01, F3): 포트가 죽어도 카드 안 작은 빨간 점 하나뿐. 노란 카드 테두리에 빨강이 묻힘.
- **대기 상태의 해결 위치가 불명확** (P03, F8/F12): "세션이 뭘 묻는 것 같은데 어디서 답하지?"

### 2. 신뢰 — 데이터가 진짜 살아있는지 확신 못 함
- **죽은 대시보드 신뢰 (최악 시나리오)** (P01, F4, 심각도 5): 서버가 죽어도 "n초 전" 숫자가 계속 올라 멈춘 화면으로 판단하게 됨.
- **"라이브"라 써놓고 안 바뀜** (P04, F19): read-screen은 실제로는 트랜스크립트. 실제 소스로 라벨링하고 시각(스탬프)을 표기해야 함.
- **보안 태세가 비가시** (P04·P05, F20): 시크릿을 값 없이 "8 keys"로만 보여주는 건 강점이지만, "로컬 전용·잠김" 표시가 화면 어디에도 없음.

### 3. 학습·온보딩 — 빈손으로 시작하면 길을 잃는다
- **제로-팀 빈 상태 부재** (P02·P03, F23): 꽉 찬 화면 기준이라 아무것도 없는 상태에서 뭘 눌러야 할지 모름. "＋ 새 팀"은 명령어만 휙 뜨고 끝.
- **CLI 설정 ↔ 대시보드 관제 분리 미교육** (P02, F23): teamctl로 만들고 대시보드에서 본다는 구조를 아무도 안 알려줌.
- **orchestrator 개념 혼동** (P02, F24), **범례가 색만 설명**(F22/F31), **완료 상태가 작업 중과 같은 초록**(F15/F7).
- **MCP 연결 상태 비노출** (P04, F25→D15): UnityMCP가 붙었는지 UI에서 알 수 없음.

### 4. 액션 — 판단은 되는데 그 자리에서 실행이 안 된다
- **확인 없는 파괴적 종료** (P04, F5): 종료 버튼이 한 방에 죽임 + 메시지·열기 바로 옆이라 오클릭 위험. 커밋 안 한 마이그레이션 중이면 치명.
- **인라인 승인/거절 없음** (P04, F8): 드로어에 맥락은 충분한데 승인 버튼이 없어 결국 세션에 들어가 y를 쳐야 함.
- **인수인계 루프가 안 닫힘** (P05, F6): 도착함에서 읽기만 되고 수락·후속·new 배지 제거가 안 됨.
- **실행 가능한 오류에 인라인 액션 부재** (P04, F21): needs-auth인데 로그인 버튼이 없어 딴 창에서 수동 처리.

### 5. 접근성 — 특정 사용자에겐 아예 벽 (심각도 최고 집중)
- **숨은 드로어가 포커스 가로챔** (P06, F1, 심각도 5): 페이지 열자마자 탭 시 안 보이는 버튼에 포커스가 잡힘 — "유령 컨트롤."
- **파괴적 동작이 스크린리더에 무음** (P07, F11/F5, 심각도 5): 종료 후 아무 안내도 안 읽힘. 죽인 건지 확인 불가.
- **대기 전이가 스크린리더에 침묵** (P07, F12, 심각도 5): 도구 핵심인 "누가 날 기다리나"가 음성 사용자에겐 침묵.
- **인수인계가 키보드/스크린리더로 도달 불가** (P06·P07, F10): 버튼이 아니라 그냥 글이라 건너뛰어짐 → 마우스 없으면 못 엶.
- **색 단독 구분 실패** (P08, F14/F15): 초록·노랑·빨강이 비슷하게 보이고 점 모양이 다 같은 동그라미. 범례도 "색=의미"라 순환논리.
- **포커스 이동/복귀·저대비·장식 글리프** (F9·F13·F28·F29): 열 때 드로어로 이동, 닫을 때 트리거로 복귀, --faint 대비 AA 이상, 장식 글리프 aria-hidden.

## 강점 — 반드시 유지할 것 (긍정 7건)

- **드로어의 "점프 없는 판단"** (P01): 지금 뭘 하는지 + 바뀐 파일까지 보여줘 안 들어가도 판단 가능.
- **세션별 산출물·diff 가시성** (P05): 클라이언트 청구 근거로 유용.
- **시각적 완성도** (P02): 정보가 많은데도 안 지저분하고 정리돼 보임 — 채택 촉진.
- **드로어의 정보 전달력** (P03): 역할 누르면 상세가 바로 떠 세션 파악 빠름.
- **키보드 토대 존재** (P06): 역할 행이 진짜 버튼이라 엔터로 열리고 포커스 테두리도 보임 — 확장만 하면 됨.
- **올바른 라벨 패턴 일부 존재** (P07): 닫기 X에 "닫기" 라벨 → 이 패턴을 전면 적용하면 됨.
- **텍스트 병기 패턴** (P08): down·needs-auth처럼 글자가 같이 있는 곳은 색약도 인지 가능 → 전 상태로 확대.

## 결정 사항 / 합의된 방향

- **접근성은 부가 기능이 아니라 P0**: 심각도 5 중 3건이 a11y. 키보드 포커스·라이브 리전·대기 호명은 출시 차단 급.
- **"관찰 → 행동" 루프를 닫는다**: 카운트·배지·인박스가 모두 클릭 가능한 진입점이 되어야 함 (F2/F6/F8/F10).
