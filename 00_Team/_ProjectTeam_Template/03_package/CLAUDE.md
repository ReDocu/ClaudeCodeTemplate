# CLAUDE.md — 패키지(Package): {팀명}

> 소속 팀: {팀명} · 이 폴더는 **배포 및 최종 선정** 역할의 작업 공간이다.
> 루트·팀 CLAUDE.md의 규칙이 이 폴더에도 적용된다.

## 임무
- **배포 및 최종 선정 작업 담당**: 개발 결과물의 검증·최종 선정, 빌드·패키징·배포 준비, 릴리스 노트 작성.
- **`../90_result_output/`의 쓰기 주체** — 완료된 작업물을 선정·패키징해 백업 보관한다.

## 접근 권한 (팀 내 상호 호환)

| 대상 | 권한 |
|---|---|
| 이 폴더 (`03_package/`) | ✅ 쓰기 — 빌드 스크립트·배포 설정·릴리스 문서·handover |
| `../00_Project/{프로젝트}/` | ✅ 접근 — 결과물 검증·빌드 (소스 코드 수정은 금지, 결함은 handover로 반송) |
| `../91_project_process/` (process 문서) | 📖 읽기만 — **갱신은 팀 관리자 전담** |
| `../11_team_doc/` | ✅ 쓰기 — 팀 공유 문서 취합 |
| `../90_result_output/` | ✅ **쓰기 주체** — 완료 백업: `YYYY-MM-DD_{프로젝트명}_{버전}/` 단위 보관 |
| 타 역할 폴더 (`02_developer` 등) | ❌ 접근 금지 |

이 폴더의 `.claude/settings.json`이 공유 폴더 접근을 허용한다. 접근 확인 프롬프트가 뜨면 위 표 범위 내에서 승인해도 된다 (규칙상 허용).

## 작업 규칙
- **메인 작업은 프로젝트·백업 폴더에서**: 검증·빌드는 `../00_Project/{프로젝트}/`에서, 최종 선정·백업은 `../90_result_output/`에서 수행한다. **이 폴더에는 보조 작업물만 둔다** — 빌드 스크립트·체크리스트 초안·검증 메모 등 역할 일반 문서와 handover.
- **세션을 마칠 때 `/handover`를 실행**해 `handover_package.md`를 갱신한다.
- **프로젝트 관련 항목에는 `[NN_프로젝트명]` 태그를 붙인다** (예: `[01_my-app] 베타 빌드 검증 완료, 90_result_output 백업`) — 팀 관리자가 이 태그로 process 문서(`../91_project_process/`)에 반영한다.
- 검증 중 발견한 결함은 「다음 할 일」에 태그와 함께 남겨 개발로 되돌린다 — **소스 직접 수정 금지**.
- **완료 백업 체크리스트** (기준: `../90_result_output/README.md`): ① 최종 산출물 ② 릴리스 문서(선정 근거 포함) ③ 필요 시 소스 스냅샷 — 대용량·독립 저장소는 태그/커밋 URL 기록으로 대체.
