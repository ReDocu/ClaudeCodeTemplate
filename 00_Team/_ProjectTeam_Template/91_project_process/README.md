# 91_project_process — 프로젝트 상태 문서 (필수 폴더 · 관리)

`00_Project/`의 각 프로젝트에 대응하는 **process 문서**를 이 폴더에서 관리한다.
프로젝트 폴더는 독립 git 저장소일 수 있어 그 안의 파일을 템플릿 저장소가 추적할 수 없다 —
그래서 상태 문서는 프로젝트 폴더 **밖**인 여기에 두고 템플릿 저장소가 추적한다.

## 규칙

- 파일명: **`NN_{프로젝트명}.md`** — `00_Project/NN_{프로젝트명}/` 폴더명과 1:1 대응 (예: `01_login-system.md`). 생성은 팀 폴더에서 `/new_project`.
- **process 문서는 프로젝트의 유일한 상태 문서**다 — 목표/완료 조건·현재 단계·진행 기록·다음 할 일·블로커. 대시보드·`/report`가 파싱하는 소스를 겸한다.
- **갱신은 팀 관리자 전담** — 팀원은 자기 `handover_{역할}.md`에 `[NN_프로젝트명]` 태그로 기록하고, 팀 관리자가 팀 `/handover`로 취합하며 반영한다 (팀원은 읽기만).
- 종료 시 제목 앞에 `[종료]`, 보류 시 `[보류]`를 붙인다. 이력은 git이 보존한다.

`01_Project01.md`는 양식 예시다 — 실제 팀에서는 `/new_team`이 복사 후 삭제한다.
