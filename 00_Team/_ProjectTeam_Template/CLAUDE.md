- **팀 생성 시점**의 커스텀 구성은 `/new_team`이 처리한다 — 역할 구성(표준/커스텀)을 확인받고 역할별 3종(CLAUDE.md·handover·settings.json)을 자동 생성하며, 이 문서의 역할 표·매트릭스도 실제 구성으로 갱신된다.
- **생성 후** 추가·변경할 때는 직접 만든다: 세션을 주는 폴더에 반드시 `CLAUDE.md`(원본: `90_Templates/CLAUDE.role.template.md`) + `handover_{역할}.md` + 공유 접근 필요 시 `.claude/settings.json` 복사 — 그리고 이 문서의 역할 표·매트릭스·작업 흐름을 함께 갱신한다.
- 폴더명은 `NN_{역할}` 권장 (예: `04_designer`). 필수 폴더(00/10/11/90)와 이름이 겹치면 안 된다.

## 프로젝트 관리 (00_Project) — 팀 관리자의 핵심 업무

- 생성: 팀 폴더에서 `/new_project {프로젝트명}` → `00_Project/NN_{프로젝트명}/process.md`.
- **`process.md`의 갱신은 팀 관리자 전담**이다. 팀원은 자기 `handover_{역할}.md`에 **`[NN_프로젝트명]` 태그**로 기록하고, 팀 관리자가 `/handover`(팀 폴더)로 취합하면서 각 process.md에 반영한다.
- 수명주기 관리(팀 관리자 전담): 단계 전환(기획→개발→패키지) · 우선순위 · 목표일 · `[보류]`/`[종료]` 표기 — 변경 시 진행 기록에 `[팀 관리자]` 한 줄을 남긴다.
- 프로젝트 폴더는 독립 git 저장소일 수 있다 — 템플릿 저장소는 `process.md`만 추적한다 (루트 .gitignore).
- 종료: 완료 조건 충족 확인 → 패키지 역할이 `90_result_output/`에 백업 → process.md 제목에 `[종료]` → 아카이브 후보.

## 작업 흐름

기획(`01_planner`) → 개발(`02_developer`) → 패키지(`03_package`).
팀원은 역할 폴더에서 세션을 열어 공유 폴더(`00_Project`/`11_team_doc`/`90_result_output`)로 작업하고, 기록은 자기 handover에 태그로 남긴다.
**메인 작업물은 `00_Project/{프로젝트}/`에, 보조 작업물(초안·메모·역할 일반 문서)은 자기 역할 폴더에 둔다** — 역할 폴더는 세션 진입점이자 보조 공간이고, 작업의 본체는 프로젝트 폴더에서 진행된다.

## 이 폴더 세션의 역할 (팀 관리자)
- **`/handover` (핵심 루틴)**: 역할 handover 3개를 읽고 → ① 태그 기반으로 각 `process.md` 갱신 ② 팀 `handover.md` 취합.
- **Builder(구현) 겸임**: 구현이 필요하면 `00_Project/{프로젝트}/`에서 직접 수행.
- 주요 명령: `/new_project` / `/handover` / `/report`(보고 + 팀 대시보드 갱신) / `/pipeline`(분석 → `11_team_doc/`).
- 역할 폴더의 작업 파일은 직접 수정하지 않는다 — 해당 역할 세션에 위임.
- 상위 폴더(루트·타 팀)의 파일에 접근하지 않는다. 전사 현황이 필요하면 사용자에게 요청한다.
