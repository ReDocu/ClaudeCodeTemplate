# 11_team_doc — 문서 취합 (필수 폴더 · 공유)

팀의 공유 문서를 취합하는 공간이다. **팀원 전원이 역할 세션에서 공유 접근**한다.

## 수명 단계 폴더 (일반 공유 문서)

문서는 **중요도가 아니라 수명 단계**로 분류한다 — 이동은 단방향(확정되면 00으로, 대체되면 90으로).

| 폴더 | 단계 | 두는 것 |
|---|---|---|
| `00_final/` | **확정본** | 타 역할이 참조하는 팀의 현재 진실 — 확정 기획서·사양·결정 기록 |
| `10_working/` | 작업 중 | 논의·검토 중 문서, 회의록 |
| `90_old/` | 대체됨 | 새 버전으로 대체된 구버전 (이력은 git이 보존하므로 정리 주기 삭제 가능) |

## 명령 산출물 (폴더 바로 아래 — 이동 금지)

- **팀 보고서 `report_{날짜}.md`** (날짜는 YYYY-MM-DD) — `/report`가 생성, `/report_root`가 수거. 수명 폴더에 넣지 않는다 (명령이 이 위치를 파싱).
- `/pipeline` 분석 결과.

## 규칙

- 문서 위치 결정 흐름: **프로젝트 산출물(메인) = `../00_Project/{프로젝트}/` → 팀 공유 확정본 = `00_final/` → 보조 작업물(초안·메모) = 내 역할 폴더**.
- 완료 백업은 `../90_result_output/`에 (패키지 역할 전담).
- 파일명에 날짜가 필요하면 `YYYY-MM-DD_{제목}.md`.
