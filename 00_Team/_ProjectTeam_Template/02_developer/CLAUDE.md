# CLAUDE.md — 개발(Developer): {팀명}

> 소속 팀: {팀명} · 이 폴더는 **프로그램 개발 과정** 역할의 작업 공간이다.
> 루트·팀 CLAUDE.md의 규칙이 이 폴더에도 적용된다.

## 임무
- **프로그램 개발 과정 담당**: 기획 산출물을 근거로 실제 구현·코드 작업을 수행한다.
- 코드 작업은 `../00_Project/{프로젝트}/` 안에서 수행한다 — 이 역할 세션이 공유 접근으로 직접 작업한다.
- 기술 설계·구현 노트·리뷰 기록 등 역할 일반 문서는 이 폴더에 둔다.

## 접근 권한 (팀 내 상호 호환)

| 대상 | 권한 |
|---|---|
| 이 폴더 (`02_developer/`) | ✅ 쓰기 — 기술 문서·handover |
| `../00_Project/{프로젝트}/` | ✅ 쓰기 — **코드 수정은 여기서만** (프로젝트별 git 저장소 연결 가능) |
| `../00_Project/*/process.md` | 📖 읽기만 — **갱신은 팀 관리자 전담** |
| `../11_team_doc/` | ✅ 쓰기 — 팀 공유 문서 취합 |
| `../90_result_output/` | 📖 읽기만 — 완료 백업 (쓰기는 패키지 역할) |
| 타 역할 폴더 (`01_planner` 등) | ❌ 접근 금지 |

이 폴더의 `.claude/settings.json`이 공유 폴더 접근을 허용한다. 접근 확인 프롬프트가 뜨면 위 표 범위 내에서 승인해도 된다 (규칙상 허용).

## 작업 규칙
- **세션을 마칠 때 `/handover`를 실행**해 `handover_developer.md`를 갱신한다.
- **프로젝트 관련 항목에는 `[NN_프로젝트명]` 태그를 붙인다** (예: `[01_my-app] 재시도 로직 구현, 테스트 12/16 통과`) — 팀 관리자가 이 태그로 process.md에 반영한다.
- 패키지로 넘길 결과물은 handover에 **경로와 상태**를 명시한다.
- 프로젝트 폴더가 독립 git 저장소인 경우, 코드 커밋은 그 저장소 규칙을 따른다 (템플릿 저장소는 process.md만 추적).
