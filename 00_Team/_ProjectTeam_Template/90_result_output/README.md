# 90_result_output — 완료 후 백업 (필수 폴더 · 보관)

**완료된 작업물의 백업 보관소**다. 프로젝트가 끝나면 **패키지 역할이 최종 선정한 작업물**을 여기에 보관한다.

## 무엇을 넣는가 (권장 3종)

1. **최종 산출물** — 빌드·배포본·완성 원고 등 "완료 조건을 충족한 결과물" 그 자체.
2. **릴리스 문서** — 릴리스 노트 또는 최종 보고서 (무엇을 왜 선정했는지 포함).
3. (필요 시) **소스 스냅샷** — 단, 대용량이거나 독립 git 저장소가 있으면 넣지 않는다. 저장소의 태그/커밋과 원격 URL을 릴리스 문서에 기록하는 것으로 대체한다.

## 규칙

- **쓰기 주체: 패키지(`03_package`) 역할** (팀 관리자 포함). 기획·개발은 읽기만.
- 보관 단위 선택: 프로젝트 완료 백업 = `YYYY-MM-DD_{프로젝트명}_{버전}/` **폴더** / 단일 문서 산출물 = `YYYY-MM-DD_{팀명}_{제목}.md` **파일**.
- 진행 중 문서는 여기 두지 않는다 — 문서 취합은 `11_team_doc/`, 작업물은 `00_Project/`.
- 여기 백업은 **영구 보관**이다 — 주기 삭제 가능한 `11_team_doc/90_old/`(대체된 구버전)와 다르다. 90번대 = 보관 (작업 00~03 → 현황 10~11 → 보관 90).
