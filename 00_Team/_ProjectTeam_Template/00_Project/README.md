# 00_Project — 진행 중 프로젝트 (필수 폴더 · 공유)

팀이 현재 담당하고 있는 프로젝트 작업물을 이 폴더에서 관리한다.
**팀원 전원(기획·개발·패키지)이 역할 세션에서 공유 접근**하는 협업 공간이다.

## 규칙

- 프로젝트 폴더명: `NN_{프로젝트명}` (영문, 번호는 생성순 고정 — 예: `01_login-system`). 생성은 팀 폴더에서 `/new_project`.
- **git 연결**: 각 프로젝트 폴더는 독립 git 저장소(실제 앱/게임 리포)로 운영할 수 있다. **템플릿 저장소는 이 폴더의 내용을 추적하지 않는다** (루트 .gitignore).
- **상태 문서는 이 폴더 밖에 있다**: 프로젝트마다 `../91_project_process/NN_{프로젝트명}.md`(process 문서)가 유일한 상태 문서이며, **갱신은 팀 관리자 전담**이다.
  팀원은 자기 `handover_{역할}.md`에 `[NN_프로젝트명]` 태그로 기록 → 팀 관리자가 `/handover`로 취합하며 반영.
- 작업물 배치: 기획·디자인 산출물, 코드, 검증 자료를 프로젝트 폴더 안에 함께 둔다. **코드 수정은 개발 역할 세션·팀 관리자만.**
- 프로젝트가 끝나면: 패키지 역할이 `90_result_output/`에 최종 백업 → 팀 관리자가 process 문서 제목에 `[종료]` → 아카이브 후보.

`01_Project01/`은 양식 예시다 — 실제 팀에서는 실제 프로젝트명으로 시작한다.
