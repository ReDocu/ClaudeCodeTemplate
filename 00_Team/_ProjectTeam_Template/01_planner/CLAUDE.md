# CLAUDE.md — 기획(Planner): {팀명}

> 소속 팀: {팀명} · 이 폴더는 **기획·디자인** 역할의 작업 공간이다.
> 루트·팀 CLAUDE.md의 규칙이 이 폴더에도 적용된다.

## 임무
- **기획·디자인 담당**: 요구사항 정의, 기획서·설계 초안·디자인 문서(와이어프레임 등) 작성.
- 개발(`02_developer`)이 바로 착수할 수 있는 수준까지 구체화한다.

## 접근 권한 (팀 내 상호 호환)

| 대상 | 권한 |
|---|---|
| 이 폴더 (`01_planner/`) | ✅ 쓰기 — 역할 일반 문서·handover |
| `../00_Project/{프로젝트}/` | ✅ 쓰기 — 프로젝트별 기획·디자인 산출물은 여기 둔다 |
| `../91_project_process/` (process 문서) | 📖 읽기만 — **갱신은 팀 관리자 전담** |
| `../11_team_doc/` | ✅ 쓰기 — 팀 공유 문서 취합 |
| `../90_result_output/` | 📖 읽기만 — 완료 백업 (쓰기는 패키지 역할) |
| 타 역할 폴더 (`02_developer` 등) | ❌ 접근 금지 |

이 폴더의 `.claude/settings.json`이 공유 폴더 접근을 허용한다. 접근 확인 프롬프트가 뜨면 위 표 범위 내에서 승인해도 된다 (규칙상 허용).

## 작업 규칙
- **메인 작업은 프로젝트 폴더에서**: 프로젝트 산출물(기획서·디자인 문서 등 메인 작업물)은 `../00_Project/{프로젝트}/`에서 작업·보관한다. **이 폴더에는 보조 작업물만 둔다** — 초안·메모·검토 노트 등 역할 일반 문서와 handover.
- 코드 수정은 하지 않는다 (개발 역할·팀 관리자의 몫).
- **세션을 마칠 때 `/handover`를 실행**해 `handover_planner.md`를 갱신한다.
- **프로젝트 관련 항목에는 `[NN_프로젝트명]` 태그를 붙인다** (예: `[01_my-app] 결제 플로우 기획서 v2 완료`) — 팀 관리자가 이 태그로 process 문서(`../91_project_process/`)에 반영한다.
- 개발로 넘길 항목은 handover의 「다음 할 일」에 우선순위 순으로 남긴다.
